# 移动端与飞书内嵌专项验收

## 范围

本轮专项验收聚焦以下关键路径：

- `/login`
- `/dashboard`
- `/okr`
- `/weekly`
- `/review`
- `/settings`

目标不是做移动端视觉重构，而是确保关键流程不被导航、滚动、表格溢出、固定区域遮挡和容器安全区问题阻断。

## 本地已完成修复

### 壳层与导航

- 为移动端新增底部导航，避免桌面侧栏隐藏后关键页面无入口
- 为壳层增加底部安全区留白，避免飞书 H5 / iOS 底部手势区遮挡内容
- 收紧头部在小屏下的信息密度，避免用户名和顶部说明挤压操作按钮

### 页面滚动与窄屏表格

- `/settings` 的“最近集成任务”和“导出日志”已改为窄屏横向可滚动
- `UserManagementPanel` 本身已保留横向滚动

### H5 视口

- 根布局已声明 `viewportFit=cover`
- 配合底部安全区 padding，降低飞书内嵌 WebView 中底部操作区被遮挡的风险

## 本地验证结果

已完成：

- `tests/mobile-feishu-ui-contract.test.ts` 通过
- `npm run build` 通过
- 关键壳层与设置页兼容修正已落地

本地验证能证明：

- 移动导航存在
- 设置页关键表格可横向滚动
- 构建产物不受兼容修正影响

本地验证不能完全证明：

- 真机小屏触控体验
- 飞书客户端内嵌 H5 的 OAuth、回跳和滚动行为
- iOS / Android WebView 下的实际安全区表现

## 当前遗留风险

- 暂无阻断当前阶段收口的遗留风险

## 外部验收结果

已完成以下外部验收闭环：

- 真机与飞书客户端内登录页可正常打开
- 真实飞书 OAuth 登录可回到系统并进入 `/dashboard`
- `/weekly` 的输入、滚动与主要按钮可正常操作
- `/review` 的导出范围选择、导出按钮与反馈提示可正常使用
- `/settings` 的关键表格与日志区域可查看，横向滚动可用
- 导出完成后的飞书文档链接可正常打开

## 最终结论

移动端与飞书内嵌专项验收已完成。本轮修复覆盖了会阻断流程的关键兼容问题，外部验收闭环已收回，当前阶段可以正式收口。

## 已完成的外部验收项

以下项目已在真机或飞书客户端中完成：

1. 使用 iPhone 或 Android 在飞书内打开系统登录页。
2. 验证 `/login` 首屏能看到“用飞书账号登录”和移动底部导航不会遮挡主要按钮。
3. 完成一次真实飞书 OAuth 登录，确认能回到 `/dashboard`。
4. 进入 `/weekly`，确认表单输入、滚动、底部按钮可达。
5. 进入 `/review`，确认导出范围选择器、导出按钮和导出结果提示在小屏下可操作。
6. 进入 `/settings`，确认“最近集成任务”“导出日志”“用户管理”可查看，横向滚动可用。
7. 触发一次季度报告导出，确认“打开飞书文档”在飞书客户端内可正常打开。
