# 真实飞书验收清单

更新时间：2026-06-19

## 1. 目的

这份清单用于把当前本地已完成的飞书接入准备，推进到真实飞书环境的最终验收。  
当前代码层面的本地验证已经完成，剩余事项主要依赖：

- 真实飞书开放平台配置
- 真实飞书账号授权
- 真实飞书日历 / 多维表格 / 云空间权限

## 2. 验收前准备

本地页面回归建议先完成以下启动检查：

- `npm run build`
- `node scripts/smoke-local.mjs`

确认本地管理员路径 `/login -> /dashboard -> /settings -> /review` 可正常打开后，再进入真实飞书验收。

在 `/settings` 页面填写并保存以下配置：

- `App ID`
- `App Secret`
- `OAuth 回调地址 redirect_uri`
- `根部门 ID root_department_id`
- `日历 ID calendar_id`
- `多维表格 App Token`
- `4 张表 Table ID JSON`
- `云空间文件夹 Token drive_folder_token`

然后把模式切换成：

- `真实飞书`

保存后，先确认设置页上方的配置检查结果：

- `OAuth 登录` 显示 `已就绪`
- `组织同步` 显示 `已就绪`
- `日历配置` 显示 `已就绪`
- `多维表格` 显示 `已就绪`
- `文档目录` 显示 `已就绪`

如果仍有 `待配置`，先不要做后续验收。

## 3. 验收顺序

按下面顺序执行，不要跳步。每一步都去设置页或对应页面看任务状态、导出日志、错误提示。

页面观察重点：

- `/settings`：看 `配置检查`、`最近集成任务`、`导出日志`
- `/review`：看 `导出范围`、导出按钮即时反馈、再回到 `/settings` 核对日志
- `/login`：确认正式飞书入口与开发入口文案清晰、失败提示可见

### 3.1 OAuth 登录

操作：

- 打开 `/login`
- 点击 `用飞书账号登录`
- 完成飞书授权

预期结果：

- 成功进入系统
- 不走 mock 登录
- 首次登录时如开启同步，应能创建或更新本地用户

失败时重点查看：

- 登录页错误提示
- `/api/v1/auth/feishu`
- `/api/v1/auth/feishu/callback`

### 3.2 组织同步

操作：

- 打开 `/settings`
- 点击 `同步组织架构`

预期结果：

- 最近集成任务里出现 `组织同步`
- 状态从 `待执行 / 执行中` 变成 `成功`
- 成功消息中能看到部门数和成员数

失败时重点查看：

- 设置页任务列表中的错误消息
- 根部门 ID 是否正确
- 飞书通讯录权限是否已开通

### 3.3 创建季度日历事件

操作：

- 打开 `/settings`
- 点击 `创建日历事件`

预期结果：

- 最近集成任务里出现 `日历事件`
- 状态为 `成功`
- 飞书日历中出现当前季度周一 / 周五重复事件

失败时重点查看：

- 设置页任务列表中的错误消息
- 日历 ID 是否正确
- 受邀范围是否配置完整

### 3.4 更新日历事件

操作：

- 在设置页调整事件时间、描述或受邀范围
- 点击 `更新日历事件`

预期结果：

- 最近集成任务里出现 `日历事件`
- 状态为 `成功`
- 飞书日历中已存在的事件被更新

失败时重点查看：

- 是否已经先执行过“创建日历事件”
- 当前季度是否已保存 event id

### 3.5 终止日历事件

操作：

- 在设置页点击 `终止日历事件`

预期结果：

- 最近集成任务里出现 `日历事件`
- 状态为 `成功`
- 飞书侧重复事件停止继续生成

失败时重点查看：

- 当前季度是否已有 event id
- 飞书侧事件是否仍存在

### 3.6 季度归档后的日历终止反馈

操作：

- 打开 `/quarters`
- 把当前季度从 `reviewing` 归档到 `archived`

预期结果：

- 前端出现归档反馈
- 如果需要终止飞书日历，反馈中能看到成功或失败信息
- 设置页导出日志可看到对应记录

失败时重点查看：

- 页面上的归档反馈
- 设置页导出日志

### 3.7 同步到多维表格

操作：

- 打开 `/settings`
- 点击 `同步到多维表格`

预期结果：

- 最近集成任务里出现 `多维表格`
- 状态为 `成功`
- 飞书多维表格的 4 张表有数据

失败时重点查看：

- App Token 是否正确
- 4 张 Table ID 是否正确
- 飞书侧写入权限是否完整

### 3.8 导出飞书文档：全公司

操作：

- 打开 `/review`
- 使用 `Super Admin`
- 导出范围选择 `全公司季度报告`
- 点击 `导出季度报告`

预期结果：

- 任务状态显示 `导出中 -> 成功`
- 返回飞书文档链接
- 设置页导出日志中范围显示 `全公司`
- 导出消息应类似：`飞书文档导出完成（全公司）`

### 3.9 导出飞书文档：指定部门

操作：

- 使用 `Super Admin`
- 导出范围选择 `指定部门季度报告`
- 选择一个目标部门
- 点击 `导出季度报告`

预期结果：

- 任务状态显示 `导出中 -> 成功`
- 设置页导出日志中范围显示 `指定部门 · <部门名>`
- 文档内容只包含目标部门相关数据

### 3.10 导出飞书文档：指定个人

操作：

- 使用 `Super Admin`
- 导出范围选择 `个人 OKR 报告`
- 选择一个目标成员
- 点击 `导出季度报告`

预期结果：

- 任务状态显示 `导出中 -> 成功`
- 设置页导出日志消息应类似：`飞书文档导出完成（个人 OKR · <成员名>）`
- 文档内容只包含该成员相关数据

### 3.11 成员导出自己的个人 OKR 报告

操作：

- 使用 `Member`
- 打开 `/review`
- 点击 `导出季度报告`

预期结果：

- 不需要再选范围或目标成员
- 成功导出自己的个人 OKR 报告
- 不能导出他人报告

## 4. 失败时请回传的信息

如果任一步失败，请优先把下面信息发回来：

- 你执行的是哪一步
- 页面上的错误提示原文
- 设置页最近集成任务中的错误原文
- 设置页导出日志中的对应记录
- 如果有飞书侧报错码，也一并发回
